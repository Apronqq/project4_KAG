### 项目名称

**医疗 KAG Agent 工作台 — 基于知识图谱与混合检索的体检辅助评估系统**

---

### 项目简介

面向体检场景构建的智能评估 Agent 系统，采用「确定性 Workflow + 自研 ReAct Agent」双层架构。用户提交自然语言体检文本后，底层 LangGraph 流水线依次完成指标解析、阈值规则判定、Neo4j 图谱多跳风险推理、混合证据检索（稠密向量 + 词汇 BM25 + Rerank 语义重排 + MMR 多样性去重）与多信号排序融合，产出结构化诊断与干预建议；上层 ReAct Agent 接管多轮追问，结合四层记忆（事实 / 诊断版本化 / 对话 / 摘要）与知识库检索工具灵活作答。系统覆盖完整的知识库管理链路、医学相关性门控防污染、三层持久化后端、自动降级与组件级健康检查。经验证，检索 MRR 0.80、P@5 0.79、nDCG@5 0.75，全链路相对纯向量基线提升 27.5%。

---

### 主要工作

**1. Agent 架构与安全边界**

- 设计并实现双层 Agent 架构：底层 LangGraph 12 节点确定性流水线作为不可绕过的安全执行引擎；上层自研 ReAct Agent 循环通过分层决策（记忆优先 → 工具检索 → 综合回答）处理多轮追问，初诊/追问路径自动分流
- ReAct 循环内置迭代上限与工具参数循环检测，全链路 7 种结构化事件流式推送，不依赖特定 LangChain 版本
- 医疗安全五层防线：确定性规则引擎 → 图谱关系推理 → Workflow 强制编排 → Agent 约束（禁止捏造诊断） → 高风险自动标注人工复核

**2. 混合检索与排序评测**

- 设计多查询规划 → 稠密向量（Milvus HNSW 复用原生距离） + 词汇检索（SQLite FTS5 BM25 持久化索引）双路 RRF 融合 → Rerank 前置语义重排 → 图谱节点重叠度 + 来源权威度多信号加权 → 质量门控 + MMR 多样性选择，全链路五步递进
- Ablation study 覆盖 5 种配置：Rerank 贡献 +11% MRR，图谱信号贡献 +9% MRR，全链路相对纯向量基线提升 27.5%；最终 MRR 0.80 / P@5 0.79 / nDCG@5 0.75

**3. 多层记忆与上下文管理**

- 四层记忆模型：事实记忆（冲突检测 + 向量化语义召回 top 5，防 prompt 膨胀）、诊断记忆（版本化存储，追问时注入最近两版做趋势对比）、对话记忆（原文摘要双字段 + 三级长度控制：单条 500 字符 / 最近 12 条 / 总 3200 字符）、摘要记忆（累计字符阈值 2000 触发 LLM 摘要生成，避免频繁调用）

**4. 知识图谱与反污染机制**

- Neo4j 医学图谱多跳关系链路（异常状态 → 疾病风险 → 疾病 → 干预/科室/复查/禁忌），Cypher 查询，双后端封装支持远程/内存自动切换
- 六层防污染纵深：文件类型白名单 → 内容哈希去重 → 文本抽取判空 → 医学相关性门控（图谱节点链接数 + 关键词密度 + 概念覆盖率三维评分，得分 < 0.15 直接拒绝入库） → 来源权威度降权（未验证文档 0.3 vs 指南 1.0） → 前端标签告警

**5. 性能优化与系统韧性**

- 图谱检索与证据检索异步并行执行，墙钟时间下降 37%；健康路径条件短路（无异常指标跳过检索/排序步骤），延迟从 ~350ms 降至 ~50ms
- Neo4j / Milvus 不可达自动降级至内存后端，降级原因透出；全链路 12+ 降级点结构化日志，6 个后端组件级健康检查含延迟检测；36 项回归测试覆盖核心链路

### 技术栈

**Agent 与编排**：自研 ReAct 循环、LangGraph StateGraph、医学相关性门控

**后端与存储**：FastAPI + SSE 异步流式、Neo4j（Cypher 多跳查询）、Milvus（HNSW 索引 + 原生距离复用）、PostgreSQL + SQLAlchemy（四表记忆模型、自动 schema 迁移）

**检索与模型**：SQLite FTS5（BM25 持久化索引）、RRF 融合、MMR 去重、DashScope Rerank/Embedding/LLM（通义千问）

**工程化**：asyncio 异步并行、结构化日志、组件级健康检查、pytest 36 项、MRR/P@5/nDCG 评测体系
